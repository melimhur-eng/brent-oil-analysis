
# Brent Oil Price Dashboard (Task 3)

Interactive dashboard for the 10 Academy Week 10 challenge — visualizes Brent oil
price history, the Bayesian change point detected in Task 2, and the researched
event dataset from Task 1.

## Project structure

```
data/               CSV outputs consumed by the API (see "Replacing placeholder data")
backend/            Flask API
frontend/           React (Vite + react-admin + MUI + Recharts) dashboard
```

## Backend (Flask)

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python app.py
```

Runs on `http://localhost:5000`.

### API endpoints

All endpoints are read-only `GET` requests returning JSON. Dates are ISO
`YYYY-MM-DD`. Errors return `{"error": "..."}` with a `400` (bad input) or
`404` (not found) status.

#### `GET /api/health`

Health check.

```json
{ "status": "ok" }
```

#### `GET /api/prices`

Daily price series with rolling mean/std of log returns and the log return itself.

| Param | Required | Default | Notes |
|---|---|---|---|
| `start` | no | full range | inclusive lower bound |
| `end` | no | full range | inclusive upper bound |
| `resample` | no | `none` | `none`, `weekly`, or `monthly` — downsamples for chart performance over long ranges |

```json
[
  { "date": "2020-01-02", "price": 66.25, "rolling_mean": -0.0004, "rolling_std": 0.0156, "log_return": 0.0031 },
  { "date": "2020-01-03", "price": 68.6, "rolling_mean": -0.0002, "rolling_std": 0.0159, "log_return": 0.0349 }
]
```

#### `GET /api/events`

Researched events (Task 1 dataset).

| Param | Required | Default | Notes |
|---|---|---|---|
| `start` / `end` | no | full range | filters on event date |
| `category` | no | all | `Geopolitical`, `Economic`, or `OPEC` |

```json
[
  { "id": 1, "date": "1990-08-02", "event": "Iraq invades Kuwait", "category": "Geopolitical", "description": "Start of the Gulf War causing oil supply concerns." }
]
```

#### `GET /api/changepoints`

Structural breaks detected by the Bayesian change point model (Task 2).

```json
[
  {
    "id": 1,
    "change_date": "2007-05-23",
    "before_mean": 0.0000308,
    "after_mean": 0.0006959,
    "pct_change": 2157.44,
    "direction": "up"
  }
]
```

#### `GET /api/summary`

Aggregate stats over a date range (defaults to the full dataset).

| Param | Required | Default |
|---|---|---|
| `start` / `end` | no | full range |

```json
{
  "start_date": "1987-05-20", "end_date": "2022-09-30",
  "start_price": 18.63, "end_price": 100.18, "pct_change": 437.73,
  "daily_volatility": 0.0221, "annualized_volatility_pct": 35.08,
  "event_count": 13, "changepoint_count": 1
}
```

#### `GET /api/events/<id>/impact`

Average price and volatility in the window before vs. after a given event — powers the dashboard's drill-down.

| Param | Required | Default |
|---|---|---|
| `window_days` | no | `30` |

```json
{
  "event": { "id": 1, "date": "1990-08-02", "event": "Iraq invades Kuwait", "category": "Geopolitical", "description": "..." },
  "window_days": 30,
  "before_avg_price": 22.26, "after_avg_price": 23.43, "pct_change": 5.24,
  "before_volatility": 0.0162, "after_volatility": 0.0180
}
```

## Frontend (React)

```bash
cd frontend
npm install
npm run dev
```

Runs on `http://localhost:5173`. Reads the API base URL from `frontend/.env`
(`VITE_API_URL`, defaults to `http://localhost:5000/api`).

Features:
- Main dashboard: price chart with change-point marker and event reference
  lines, date range selector (text inputs + chart brush), category filter,
  key indicator cards (latest price, % change, annualized volatility, change
  points detected), and a clickable event list with drill-down impact stats.
- `Events` and `Change Points` resource pages (list/filter/search/show) via
  react-admin, for exploring the full underlying data.

## Replacing placeholder data

`data/prices.csv` was generated by `data/generate_prices.py` as a **placeholder**
daily Brent price series (1987-05-20 to 2022-09-30), anchored to approximate
historical annual averages, since the real Task 1/2 notebook wasn't available at
dashboard build time. Before final submission, replace it with the actual
`log_returns.csv` export from the Task 2 notebook, renamed to `data/prices.csv`
with columns `Date,Price,Rolling_Mean,Rolling_STD,Log_Return`.

`data/changepoints.csv` (`Change_Date,Before_Mean,After_Mean`) and
`data/events.csv` (`Date,Event,Category,Description`) already hold the real
outputs from Task 1/2 — extend `changepoints.csv` with more rows if the model
detects multiple switch points.

No backend code changes are needed to swap in real data — just overwrite the
CSVs and restart Flask (or call `data_loader.reload()`).
